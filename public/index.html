EasyExtract

Setup Required

EasyExtract needs API keys to process documents. Edit .dev.vars in the project root, then restart wrangler dev.

Step-by-step setup guide

1. OpenAI API Key

  1. Go to platform.openai.com/api-keys
  2. Click "Create new secret key"
  3. Copy it and paste into .dev.vars as OPENAI_API_KEY=sk-...

2. Google Cloud Vision API

  1. Go to Cloud Vision API and click Enable
  2. Go to Service Accounts
  3. Create a service account (or use an existing one)
  4. Click into the service account → Keys tab → Add Key → JSON
  5. Open the downloaded JSON file, copy the entire contents onto one line

3. Fill in .dev.vars

OPENAI_API_KEY=sk-your-key-here
GCP_SA_KEY={"type":"service_account","project_id":"...","private_key":"...","client_email":"...","token_uri":"https://oauth2.googleapis.com/token"}
GCP_PROJECT_ID=your-project-id

4. Restart the dev server

npm run dev

Settings

Extraction Instructions Runs on every document

Sent to the AI each time it reads a document and fills in your template fields. Use {{fields}} as a placeholder — EasyExtract replaces it with the field list from your template automatically. If you leave out {{fields}}, the field list is appended at the end.

OCR Performance

Controls how many pages are sent per Google Vision OCR request. Lower values can reduce per-request payload size; higher values can improve throughput.

Recommended starting point: 8 for faster first results on large files, or 16 for maximum throughput on stable connections.

Template Builder Instructions Used in AI template builder

Guides the AI when you click "Build with AI" to generate a new template from a plain-text description. Adjust naming conventions, field style, or add domain-specific rules — for example, always use short titles or always mark revenue fields as required.
